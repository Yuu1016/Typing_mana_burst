========================================================================
【フロントエンド開発ロードマップ】タイピング・マナ・バースト
========================================================================
React (TypeScript / Vite) を用いたフロントエンド実装のステップ・バイ・ステップ手順。
バックエンド（Spring Boot）のAPIと連携し、「触り心地」の良いWebゲームを構築する。

------------------------------------------------------------------------
■ 基本開発方針
------------------------------------------------------------------------
1. 静的モックから動的ロジックへ
   - まずはAPIを繋がず、ダミーデータを使って「画面の見た目（CSS）」を完成させる。
2. ロジックの分離（カスタムフック）
   - タイピング判定やコスト計算などの複雑なロジックはコンポーネント（UI）に直接書かず、`useBattle.ts` などのカスタムフックに分離してテストしやすくする。
3. 最後に「演出」を乗せる
   - ゲームの根幹が動いてから、CSSアニメーションや画面シェイクなどのリッチな演出を追加する。

========================================================================
■ 詳細開発ステップ（全6段階）
========================================================================

Step 1: プロジェクト設定とルーティング基盤の構築
------------------------------------------------------------------------
【概要】
Viteを用いてReact+TypeScriptプロジェクトを立ち上げ、全5画面の画面遷移（ルーティング）を作成する。

【実装タスク】
[ ] Vite + React + TypeScript プロジェクトの作成 (`npm create vite@latest`)
[ ] React Router のインストール (`npm install react-router-dom`)
[ ] 各画面の「空のコンポーネント」を作成
  - `Title.tsx` (タイトル画面)
  - `Home.tsx` (ホーム画面)
  - `Deck.tsx` (編成画面)
  - `Battle.tsx` (バトル画面)
  - `Upgrade.tsx` (強化画面)
[ ] `App.tsx` にルーティングを設定し、画面間のリンク（画面遷移）が動くことを確認する。

【💡 コミット目安】
`chore: Reactプロジェクトの立ち上げと基本ルーティング設定`


Step 2: 静的レイアウト・UIコンポーネントの作成
------------------------------------------------------------------------
【概要】
APIやタイピングロジックを一旦忘れ、CSS（またはTailwind等）を使って企画書通りの画面レイアウトをダミーデータで組む。

【実装タスク】
[ ] 共通パーツの作成（ボタン、ヘッダー、ステータスバー等）
[ ] バトル画面（`Battle.tsx`）の静的レイアウト構築 ★最優先
  - 上下分割、敵/味方ステータスバー
  - 中央のタイピング文字表示エリア
  - 下部の手札（5枚）並びとマナゲージ
[ ] その他画面のレイアウト（ホーム、編成、工房）
  - 編成画面はグリッドレイアウト（CSS Grid）でカードを並べる。

【💡 コミット目安】
`feat: 全画面の静的レイアウトとUIコンポーネントを作成`


Step 3: タイピングとバトルロジックの実装（カスタムフック）
------------------------------------------------------------------------
【概要】
ゲームのコアとなる「キーボード入力判定」と「コスト計算」を実装する。まずはバックエンド通信なしでローカルで動かす。

【実装タスク】
[ ] `useTyping.ts` (カスタムフック) の作成
  - 画面全体の `keydown` イベントをリッスンする。
  - 出題ワードに対して「正しいキーを押したか」「ミスしたか」を判定し、Stateを更新する。
[ ] `useBattle.ts` (カスタムフック) の作成
  - スキルカードを選択した際の「コスト加算」処理。
  - コストが上限に達したかの判定（JUST判定）。
[ ] バトル画面のUIにフックを接続し、キーボードを叩いて文字が消えていく挙動を確認。

【💡 コミット目安】
`feat: タイピング判定とコスト計算ロジック(useTyping, useBattle)を実装`


Step 4: バックエンドAPIとの結合
------------------------------------------------------------------------
【概要】
作成済みの Spring Boot API を呼び出し、ゲームに本物のデータを流し込む。

【実装タスク】
[ ] API通信用のユーティリティ関数（または `axios` の設定）を作成。
[ ] `MasterDataController` との結合
  - アプリ起動時（またはタイトル画面）に `/api/v1/master-data` を叩き、全スキル/ステージ情報を取得・保存。
[ ] `UserController` との結合
  - ホーム画面で `/api/v1/users/{id}` を叩いてHPや所持ゴールドを表示。
  - 編成画面でスロット入れ替えAPI（PUT）を呼び出す。
  - 工房画面で強化API（POST）を呼び出す。
[ ] `BattleController` との結合
  - バトル開始時（`/start`）、スキル発動時（`/cast`）、決着時（`/result`）のAPIを繋ぎ、実際のHP増減を画面に反映させる。

【💡 コミット目安】
`feat: バックエンドAPIとの結合（マスタ取得、編成、バトル進行）`


Step 5: 「触り心地」の追求（アニメーションと演出）
------------------------------------------------------------------------
【概要】
企画書に記載された「Webゲームとしての触り心地」を CSS や React の状態を使って実装する。

【実装タスク】
[ ] コスト「ピッタリボーナス」の演出
  - `currentCost === limitCost` の時に専用の CSS クラス（発光・明滅）を付与する。
[ ] タイピング成功時のフィードバック
  - 正解キー入力時に、画面全体に数ミリ秒 `.shake`（画面揺れ）クラスを付与する。
[ ] 魔法発動（キャスト）時のド派手な演出
  - API呼び出し中（待機中）に画面を暗転させ、光の帯やエフェクト（CSSアニメーション）を走らせる。
[ ] ドット絵風のUIホバーエフェクト（カーソルアイコンのアニメーション等）を追加。

【💡 コミット目安】
`feat: タイピング時・魔法発動時のCSSアニメーションと演出を追加`


Step 6: 最終調整とデバッグ
------------------------------------------------------------------------
【概要】
実際にゲームを通しプレイし、違和感の解消やバグ修正を行う。

【実装タスク】
[ ] タイピング単語リストの拡充（長文や記号なしなどの調整）。
[ ] バックエンド側で返ってきたエラー（コスト不足など）を、フロント画面上で「エラーメッセージ」として綺麗にポップアップ表示する処理。
[ ] （任意）フリー音源サイトから取得した効果音（SE）やBGMを組み込む。

【💡 コミット目安】
`fix: 全体的なバランス調整、バグ修正、およびSEの追加`
