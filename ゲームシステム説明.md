# 【仕様書】タイピング・マナ・バースト（Typing Mana Burst）

## 1. ゲーム概要
本作は、魔法の詠唱（タイピング）を通じて敵と戦う「ターン制アクションRPG」である。
プレイヤーの「タイピング速度」と「正確性」が直接キャラクターの攻撃力・防御力・リソース回復に直結する、新感覚のバトルシステムを特徴とする[cite: 4]。

## 2. 開発環境・技術スタック
*   **フロントエンド:** React (TypeScript / Vite)[cite: 4]
*   **バックエンド:** Java (Spring Boot / Spring Data JPA)[cite: 4]
*   **データベース:** PostgreSQL[cite: 4]
*   **インフラ:** Docker (マルチコンテナ構成)[cite: 4]

## 3. 実装済みのコア・バトルシステム（フェーズ進行）
バトルは以下の3つのフェーズを繰り返して進行する。

### ① SELECT（魔法選択フェーズ）
*   毎ターン、手札から発動したい魔法（スキル）を選択する。
*   **マナの成長:** 毎ターン、マナの最大上限が1ずつ増加する（最大10まで）。
*   **オーバーキャスト（限界詠唱）:** 現在のマナ（コスト）が足りない魔法も強引に選択できる。ただしその場合、詠唱ワードが「2単語（文字数2倍）」になり、詠唱時間が伸びるハイリスク・ハイリターンな仕様となっている。

### ② ATTACK（詠唱・攻撃フェーズ）
*   魔法を選択した瞬間、制限時間（タイマー）が作動する。
*   制限時間内に、魔法のコストに応じたランダムな英単語をタイピングする。
*   **ミスペナルティ:** タイポ（ミス）が「10回」に達すると集中力が切れ、詠唱失敗（不発）となる。
*   **時間切れ:** 制限時間内に打ち切れなかった場合も不発となる。
*   成功すると敵にダメージを与え、防御フェーズへ移行する。

### ③ DEFENSE（防衛・マナチャージフェーズ）
*   魔法発動後、画面が赤くなり「5秒間」の防衛フェーズが開始される。
*   次々と出現する防衛ワード（`BLOCK`, `SHIELD` など）を猛スピードでタイピングする。
*   **マナチャージ:** 単語を1つ打ち切るごとに、次ターンのマナが「+1」追加回復する。
*   **ダメージカット（ガード）:** 単語を1つ打ち切るごとに、直後に来る敵からのダメージを「-3」軽減する。完全にダメージを0に抑え込むと「PERFECT GUARD」となる。

## 4. リザルト・報酬システム
*   敵のHPを0にすると勝利（VICTORY）となり、バックエンドAPIを通じてDBに「クリアターン数」「タイピング成績」が保存される。
*   報酬としてゴールドを獲得でき、今後の強化に使用できる。
